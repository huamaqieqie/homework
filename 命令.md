# LeWM 服务器命令

本文档专门针对当前 LeWM 项目路径：

```bash
/data1/Johnny/challenge/wrf/homework
```

通用 JEPA 可视化工具说明见：

```text
tools/jepa_viz/COMMANDS.md
```

## 0. 环境准备

```bash
source /data1/Johnny/challenge/wrf/.venv/bin/activate

export SERVER_REPO=/data1/Johnny/challenge/wrf/homework
cd $SERVER_REPO
source env_cache.sh

export JEPA_VIZ_DIR=$SERVER_REPO/tools/jepa_viz
export JEPA_VIZ_RUN_NAME=$(date +%Y%m%d_%H%M%S)
export JEPA_VIZ_OUTPUT_ROOT=$JEPA_VIZ_DIR/output/$JEPA_VIZ_RUN_NAME

echo "JEPA_VIZ_OUTPUT_ROOT=$JEPA_VIZ_OUTPUT_ROOT"
```

确认不会写到 `/root`：

```bash
echo "LEWM_OUTPUT_ROOT=$LEWM_OUTPUT_ROOT"
echo "HOME=$HOME"
echo "XDG_CACHE_HOME=$XDG_CACHE_HOME"
echo "STABLE_PRETRAINING_CACHE_DIR=$STABLE_PRETRAINING_CACHE_DIR"
echo "STABLEWM_HOME=$STABLEWM_HOME"
echo "TMPDIR=$TMPDIR"
```

## 1. 训练 LeWM

```bash
cd $SERVER_REPO/le-wm

python train.py \
  data=pusht \
  output_model_name=lewm_15 \
  subdir=my_run \
  data.dataset.name=/manifoldai-training/johnny/challenge/lewm-pusht/datasets/pusht_expert_train.h5 \
  trainer.max_epochs=15 \
  loader.batch_size=128

cd $SERVER_REPO
```

训练启动后，在终端日志中找到类似这一行：

```text
[RegistryLogger] run_id=... run_dir=... sidecar.json + summary.json + metrics.csv will live here
```

记下其中的 `run_dir`，后面画训练曲线会用到：

```bash
export RUN_DIR=<训练日志里的run_dir>
```

## 2. 监控训练曲线

一次性生成曲线：

```bash
export RUN_DIR=<训练日志里的run_dir>

MODE=plot \
TRAIN_LOG=$RUN_DIR/metrics.csv \
TRAINING_OUT=$JEPA_VIZ_OUTPUT_ROOT/training \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

训练过程中持续刷新：

```bash
export RUN_DIR=<训练日志里的run_dir>

MODE=watch \
TRAIN_LOG=$RUN_DIR/metrics.csv \
TRAINING_OUT=$JEPA_VIZ_OUTPUT_ROOT/training \
WATCH_INTERVAL=60 \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

## 3. 评估 LeWM

把 `POLICY` 替换成你的 checkpoint 路径或相对 `STABLEWM_HOME` 的 policy 路径：

```bash
cd $SERVER_REPO/le-wm

python eval.py \
  --config-name=pusht.yaml \
  policy=<你的checkpoint路径或相对STABLEWM_HOME的policy路径> \
  seed=42 \
  eval.num_eval=50 \
  eval.eval_budget=300

cd $SERVER_REPO
```

## 4. 导出 LeWM Latents

示例 checkpoint：

```bash
export CKPT_PATH=$SERVER_REPO/outputs/stable-wm/checkpoints/lewm_15/weights_epoch_15.pt
```

如果你的 checkpoint 在别的位置，改成真实路径：

```bash
export CKPT_PATH=<你的checkpoint路径>
```

导出验证集 latent：

```bash
MODE=export_latents \
DATA_CONFIG=pusht \
DATASET_NAME=/manifoldai-training/johnny/challenge/lewm-pusht/datasets/pusht_expert_train.h5 \
LATENT_CONFIG=$SERVER_REPO/le-wm/config/train/lewm.yaml \
LATENT_CHECKPOINT=$CKPT_PATH \
LATENT_SPLIT=val \
LATENT_MAX_SAMPLES=1024 \
LATENT_OUT=$JEPA_VIZ_OUTPUT_ROOT/latents \
BATCH_SIZE=128 \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

最小导出测试：

```bash
MODE=export_latents \
DATA_CONFIG=pusht \
DATASET_NAME=/manifoldai-training/johnny/challenge/lewm-pusht/datasets/pusht_expert_train.h5 \
LATENT_CONFIG=$SERVER_REPO/le-wm/config/train/lewm.yaml \
LATENT_CHECKPOINT=$CKPT_PATH \
LATENT_SPLIT=val \
LATENT_MAX_SAMPLES=16 \
LATENT_OUT=$JEPA_VIZ_OUTPUT_ROOT/latents_test \
BATCH_SIZE=8 \
NUM_WORKERS=0 \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

输出文件：

```text
tools/jepa_viz/output/<YYYYMMDD_HHMMSS>/latents/
  latents.npz
  metadata.jsonl
  summary.json
```

## 5. 可视化 LeWM Latents

```bash
LATENT_DIR=$JEPA_VIZ_OUTPUT_ROOT/latents \
LATENT_VIZ_OUT=$JEPA_VIZ_OUTPUT_ROOT/latent_viz \
LATENT_COLOR_BY=action \
LATENT_MAX_POINTS=5000 \
LATENT_ALIGNMENT_COUNT=4 \
LATENT_ACTIVE_THRESHOLD=1e-2 \
LATENT_ACTIVE_RELATIVE_THRESHOLD=0.0 \
LATENT_PAIRWISE_DENSITY=1 \

LATENT_NN_QUERIES=8 \
LATENT_TOP_K=5 \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

## 6. LeWM Prediction 能力可视化

这一步使用第 4 节导出的 latent，分析 `z_pred` 和 `z_target` 的预测对齐能力。

```bash
MODE=visualize_prediction \
LATENT_DIR=$JEPA_VIZ_OUTPUT_ROOT/latents \
PREDICTION_VIZ_OUT=$JEPA_VIZ_OUTPUT_ROOT/prediction_viz \
PREDICTION_GROUP_BY=action \
PREDICTION_MAX_GROUPS=8 \
PREDICTION_ACTION_BINS=4 \
PREDICTION_INTERVAL=std \
PREDICTION_HEATMAP_VMIN_QUANTILE=0.05 \
bash $JEPA_VIZ_DIR/run_jepa_viz_template.sh
```

输出目录：

```text
tools/jepa_viz/output/<YYYYMMDD_HHMMSS>/prediction_viz/
```

常见输出：

```text
target_pred_cosine_vs_horizon_by_*.png
target_pred_cosine_boxplot_by_*.png，如果只有一个 horizon
target_pred_cosine_vs_horizon_by_*.csv
target_pred_alignment_heatmap.png
target_pred_alignment_heatmap.csv
target_pred_alignment_heatmap_metrics.json
action_norm_vs_cosine_scatter.png
action_norm_bin_vs_cosine_boxplot.png
action_norm_bin_vs_mse_boxplot.png
<action_component>_bin_vs_cosine_boxplot.png
<action_component>_bin_vs_mse_boxplot.png

prediction_report.json
```

如果当前 latent 没有 rollout、condition ablation 或 goal latent 对应字段，下面这些图会跳过，原因会写进 `prediction_report.md`：

```bash
MODE=all \

LATENT_CONFIG=$SERVER_REPO/le-wm/config/train/lewm.yaml \

LATENT_MAX_SAMPLES=1024 \
LATENT_OUT=$JEPA_VIZ_OUTPUT_ROOT/latents \
LATENT_DIR=$JEPA_VIZ_OUTPUT_ROOT/latents \
LATENT_VIZ_OUT=$JEPA_VIZ_OUTPUT_ROOT/latent_viz \
PREDICTION_VIZ_OUT=$JEPA_VIZ_OUTPUT_ROOT/prediction_viz \
LATENT_COLOR_BY=action \

PREDICTION_GROUP_BY=action \

```

```text
export_latents
```
